# LLM Wiki — Schema (위키 규칙)

이 파일은 Claude가 wiki를 생성하고 유지할 때 따라야 할 규칙을 정의합니다.
**이 파일은 사람이 직접 수정합니다. Claude는 이 파일을 수정하지 않습니다.**

---

## 위키의 목적

연구 논문과 학술 자료를 체계적으로 정리하는 개인 지식 베이스.
새로운 논문을 읽을 때마다 위키를 업데이트하여 지식이 누적되고 연결되도록 한다.

---

## 디렉토리 구조

```
sources/      # 원본 자료 (논문 PDF, 요약 텍스트 등) — 절대 수정 금지
wiki/         # Claude가 생성/관리하는 마크다운 페이지
schema/       # 사람이 정의한 규칙 (이 폴더)
```

## Wiki 페이지 구조

### 필수 페이지 목록

| 파일명 | 설명 |
|--------|------|
| `wiki/index.md` | 전체 위키 목차 및 네비게이션 |
| `wiki/papers/` | 논문별 상세 페이지 (1논문 1파일) |
| `wiki/topics/` | 주제별 종합 페이지 |
| `wiki/authors/` | 저자별 연구 동향 |
| `wiki/glossary.md` | 핵심 용어 정의 |
| `wiki/timeline.md` | 연구 흐름 타임라인 |

### 논문 페이지 형식 (`wiki/papers/<slug>.md`)

```markdown
# 논문 제목

- **저자**: 
- **연도**: 
- **출처**: (학회/저널명)
- **링크**: 
- **태그**: #태그1 #태그2

## 핵심 기여 (Contribution)
논문이 해결한 문제와 주요 기여를 2-4문장으로.

## 방법론 (Method)
핵심 아이디어와 접근 방식.

## 결과 (Results)
주요 실험 결과와 수치.

## 한계 (Limitations)
저자가 언급한 한계 또는 내가 파악한 한계.

## 관련 논문
- [[논문슬러그]] — 연결 이유
```

### 주제 페이지 형식 (`wiki/topics/<topic>.md`)

```markdown
# 주제명

## 개요
이 주제의 핵심 개념과 연구 방향.

## 주요 논문
- [[논문슬러그]] (연도) — 한줄 요약

## 핵심 개념
용어와 개념 정리.

## 미해결 문제 (Open Problems)
현재 연구계에서 풀리지 않은 문제들.

## 관련 주제
- [[다른주제]] — 연결 이유
```

## Claude 작업 규칙

### Ingest (새 논문 추가)
새 소스를 추가하면:
1. `wiki/papers/<slug>.md` 생성
2. 관련 `wiki/topics/*.md` 업데이트 (해당 논문 추가)
3. 새 용어가 있으면 `wiki/glossary.md` 업데이트
4. `wiki/index.md` 업데이트
5. `wiki/timeline.md` 업데이트

### Query (질문에 답하기)
- 위키 파일들을 먼저 읽고 답변
- 답변 끝에 참고한 위키 페이지 명시
- 위키에 없는 내용이면 "위키에 없음"이라고 솔직하게 답변

### Lint (위키 점검)
주기적으로 또는 요청 시:
- 모순되는 내용 찾기
- 링크 끊김 확인
- 업데이트가 필요한 오래된 내용 표시
- 중복 내용 통합 제안

## 태그 규칙

논문 태그는 아래 카테고리에서 선택:

**분야**: `#NLP` `#CV` `#RL` `#ML` `#DL` `#multimodal`
**방법론**: `#transformer` `#diffusion` `#GAN` `#RLHF` `#RAG` `#agent`
**주제**: `#LLM` `#alignment` `#reasoning` `#generation` `#retrieval`
**상태**: `#읽음` `#읽는중` `#읽을예정` `#중요`
